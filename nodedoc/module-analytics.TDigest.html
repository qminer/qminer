analytics.TDigest

Source: analyticsdoc.js:2151

TDigest is a methods that approximates the CDF function of streaming measurements. Data structure useful for percentile and quantile estimation for online data streams. It can be added to any anomaly detector to set the number of alarms triggered as a percentage of the total samples. Adding new samples to the distribution is achieved through partialFit and querying the model (computing quantiles) is implemented by predict function.
This is based on the Data Lib Sketch Implementation: t-digest.js
Paper: Ted Dunning, Otmar Ertl - Computing Extremely Accurate Quantiles Using t-Digests.

new TDigest([arg])

TDigest quantile estimation on streams

Example 

// import modules
var qm = require('qminer');
var analytics = qm.analytics;
// create the default TDigest object
var tdigest = new analytics.TDigest();
// create the data used for calculating quantiles
var inputs = [10, 1, 2, 8, 9, 5, 6, 4, 7, 3];
// fit the TDigest model
for (var i = 0; i < inputs.length; i++) {
    tdigest.partialFit(inputs[i]);
}
// make the prediction for the 0.1 quantile
var prediction = tdigest.predict(0.1);
// save the model
tdigest.save(fs.openWrite('tdigest.bin')).close();
// open the tdigest model under a new variable
var tdigest2 = new analytics.TDigest(fs.openRead('tdigest.bin'));

predict(x) → number

Returns a quantile given input number, that is the approximate fraction of samples smaller than the input (0.05 means that 5% of data is smaller than the input value).
